## 📋 测试流程

### 阶段一：开发环境自测（必做）

#### 1. 代码编写完成后
- [ ] 检查是否有明显的逻辑错误
- [ ] 确认所有 import 路径正确
- [ ] 确认 TypeScript 类型定义完整
- [ ] 运行 `npm run lint` 无报错

#### 2. 本地构建测试
```bash
# 1. 重新生成 Bundle
npx react-native bundle --platform android --dev false --entry-file index.js --bundle-output android/app/src/main/assets/index.android.bundle --assets-dest android/app/src/main/res/

# 2. 构建 Release APK
cd android && ./gradlew assembleRelease

# 3. 确认构建成功
ls -lh app/build/outputs/apk/release/app-release.apk
```

#### 3. 模拟器测试
```bash
# 1. 安装到模拟器
adb install -r app/build/outputs/apk/release/app-release.apk

# 2. 启动应用
adb shell am start -n com.divination.bazi/.MainActivity

# 3. 监控日志
adb logcat | grep -i "ReactNativeJS\|FATAL\|CRASH"
```

## 🚀 发布流程

### 正式发布步骤

1. **本地测试通过**
   ```bash
   # 构建 Release
   cd android && ./gradlew clean assembleRelease
   
   # 安装测试
   adb install -r app/build/outputs/apk/release/app-release.apk
   
   # 运行完整测试清单
   # 确认所有功能正常
   ```

2. **提交代码**
   ```bash
   git add .
   git commit -m "feat: 描述新功能"
   git push origin master
   ```

3. **创建 Release**
   ```bash
   git tag v1.0.1
   git push origin v1.0.1
   ```

4. **GitHub Actions 自动发布**
   - 访问 https://github.com/joe188/divination-app/actions
   - 确认构建成功
   - 下载 APK 测试
   - 确认 Release 说明完整

5. **真机测试**
   - 下载最新 APK
   - 安装到真机
   - 运行核心功能测试
   - 确认无问题后发布
